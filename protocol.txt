Format for client:
    <SESSION_TOKEN>/<COMMAND>/<JSON_ARGUEMENTS>

The session token can be left out if the user is making an account or logging in:
    /<COMMAND>/<JSON_ARGUEMENTS>

Server response:
    <RESPONSE_CODE>/<JSON_DATA>

Server response codes: 
    NOT_AUTHORIZED = 401
    AUTHORIZED = 201
    BAD_REQUEST = 400
    SERVER_FAILED = 500
    SUCCESS = 200

NOT_AUTHORIZED returns when the session_token isn't present or is invalid
AUTHORIZED returns the newly generated session key when the user logs into a new session
